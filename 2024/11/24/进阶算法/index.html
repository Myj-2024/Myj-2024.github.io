回溯算法

一、数独求解

数独求解

数独求解是回溯算法的经典应用，通过逐步填充空格并验证，在不满足约束条件时及时回溯，最终找到数独问题的解决方案。

时间复杂度:最差 O(N^m) 空间复杂度:O(m)

算法讲解

介绍

数独（Sudoku）是一种基于逻辑的组合数字放置谜题，目标是在一个 9×9 的网格中填入数字 1-9，使得每行、每列和每个 3×3 子网格中的数字不重复。数独求解算法通常采用回溯搜索的策略，结合多种约束传播技术提高效率。数独问题是一个 NP 完全问题，意味着没有已知的多项式时间算法可以解决所有情况，但通过合理的启发式和优化手段，可以高效解决绝大多数数独谜题。

数独求解算法不仅在游戏解谜中有应用，也是约束满足问题（Constraint Satisfaction Problem，CSP）的经典案例，在人工智能、优化算法研究等领域有重要价值。

算法步骤

  1. 输入一个9×9的数独谜题，空白格用0或’.’表示
  2. 找到一个空白格（未填数字的位置）
  3. 尝试在该位置填入1-9中的一个数字
  4. 检查填入的数字是否满足数独规则（行、列、3×3子网格不重复）
  5. 如果满足规则，递归地解决剩余空白格
  6. 如果递归求解成功，则数独解决
  7. 如果递归求解失败或当前数字不满足规则，回溯并尝试下一个数字
  8. 如果1-9都尝试完仍未解决，则回溯到上一个空白格
  9. 如果不存在空白格，则数独已解决

核心特性

  • 回溯算法：通过深度优先搜索和回溯尝试所有可能的组合
  • 约束传播：利用数独规则减少搜索空间
  • 时间复杂度：最坏情况为O(9^m)，m是空白格数量
  • 空间复杂度：O(m)，主要用于递归调用栈
  • 确定性：对于有唯一解的数独，算法总能找到解
  • 适用范围：标准9×9数独及其变种

二、全排列生成

全排列生成

全排列生成是回溯算法的典型应用，通过逐个尝试所有可能的元素组合，生成指定元素集合的所有可能排列。

时间复杂度:最差 O(n!) 空间复杂度:O(n)

算法讲解

介绍

全排列（Permutation）是一种经典的组合问题，指的是将一组元素（通常是数字或字符）重新排列，得到它们所有可能的排列方式。对于包含n个不同元素的集合，共有n!（n的阶乘）种不同的排列方式。

回溯算法（Backtracking）是解决全排列问题的理想方法，它通过递归尝试所有可能的选择，并在需要时撤销选择（回溯），系统性地生成所有可能的排列。

全排列问题是理解回溯算法非常合适的案例，而且它应用广泛，比如组合优化、密码学、图论等领域。

算法步骤

回溯法解决全排列问题的基本步骤：

  1. 选择一个起始位置（初始为0）
  2. 对于当前位置，尝试放置每个还未使用的元素
  3. 放置一个元素后，递归处理下一个位置
  4. 递归结束后，撤销当前选择（回溯），尝试其他可能性
  5. 当所有位置都已填满元素时（到达叶节点），记录当前排列

核心特性

  • 递归回溯：通过递归试探和回退，系统地生成所有排列
  • 状态恢复：每次探索完成后恢复状态，确保不重复使用元素
  • 时间复杂度：O(n!)，与排列数量成正比
  • 空间复杂度：O(n)，主要是递归调用栈的开销
  • 适用范围：中小规模的排列问题（元素较多时计算量爆炸）

三、N皇后问题

N皇后问题

N皇后问题是回溯算法的经典应用，要求在N×N的棋盘上放置N个皇后，使得它们互不攻击（不在同一行、同一列或同一对角线上）。

时间复杂度:最差 O(N!) 空间复杂度:O(N)

算法讲解

介绍

N皇后问题（N-Queens Problem）是一个经典的组合优化问题，源于国际象棋。问题要求在N×N的棋盘上放置N个皇后，使得任意两个皇后都不能互相攻击，即不能处于同一行、同一列或同一对角线上。

回溯算法（Backtracking）是解决N皇后问题的标准方法。该算法通过尝试在棋盘上放置皇后，当发现当前放置方案无法继续时，就撤销最近的选择，回溯到上一步并尝试其他可能性，直到找到完整解或尝试所有可能后确认无解。

N皇后问题不仅是理解回溯算法的典型案例，也在计算机科学、人工智能、约束满足问题研究中具有重要意义。

算法步骤

使用回溯算法解决N皇后问题的基本步骤：

  1. 从第一行开始，尝试在每一行放置一个皇后
  2. 对于当前行，尝试在每一列放置皇后
  3. 检查放置是否有效（不与已放置的皇后冲突）
  4. 如果有效，则递归处理下一行
  5. 如果无法在当前行找到有效位置，则回溯到上一行，尝试其他列位置
  6. 当成功放置N个皇后时，记录解决方案
冲突检测
  1. 检查同一列是否已有皇后
  2. 检查左上到右下对角线是否已有皇后
  3. 检查右上到左下对角线是否已有皇后

核心特性

  • 递归回溯：通过递归尝试不同放置方案，失败则回溯
  • 剪枝：提前检测无效放置，避免无谓的搜索
  • 时间复杂度：最坏情况下为O(N!)，实际会因剪枝而降低
  • 空间复杂度：O(N)，存储棋盘状态和递归调用栈
  • 约束满足：每步决策都必须满足所有已有皇后不受攻击的约束

动态规划

一、最长递增子序列 (LIS)

最长递增子序列 (LIS)

最长递增子序列是指在一个给定的数字序列中，找到一个子序列，使得这个子序列元素单调递增，并且长度尽可能大。该问题可以使用动态规划高效解决。

时间复杂度:最佳 O(n²) | 平均 O(n²) | 最差 O(n²) 空间复杂度:O(n)

应用场景/前提条件
  • 动态规划
  • 数组
  • 递增序列

算法讲解

介绍

最长递增子序列（Longest Increasing Subsequence, LIS）是一个经典的动态规划问题，目标是找出一个给定序列中最长的子序列，要求该子序列中的所有元素按照严格递增的顺序排列。注意，子序列不要求连续，但要保持原序列中元素的相对顺序。

最长递增子序列问题可以通过多种方法解决，其中动态规划是最为经典和直观的方法，时间复杂度为O(n²)，n是序列长度。还有使用二分查找优化的方法可以将时间复杂度降低到O(n log n)。

算法步骤

  1. 定义状态：dp[i]表示以第i个元素结尾的最长递增子序列的长度
  2. 初始化：所有dp[i]初始值为1（至少包含自身一个元素）
  3. 状态转移：对于每个位置i，检查所有在它之前的位置j：
    • 如果nums[i] > nums[j]，则可以将nums[i]接在以nums[j]结尾的子序列之后
    • dp[i] = max(dp[i], dp[j] + 1)
  4. 结果：max(dp[0], dp[1], …, dp[n-1])

斐波那契数列是一个经典的数列，其中每个数是前两个数的和。这个问题可以用递归、记忆化搜索和动态规划等多种方法解决，是理解动态规划的基础案例。

算法讲解

介绍

斐波那契数列（Fibonacci Sequence）是一个经典的数学序列，它的定义很简单：前两个数是0和1（或1和1，取决于从哪个索引开始），此后的每个数都是前两个数的和。形式上，斐波那契数列可以表示为：F(0) = 0, F(1) = 1, F(n) = F(n-1) + F(n-2) (n ≥ 2)。

动态规划（Dynamic Programming）是求解斐波那契数列的高效方法，核心思想是通过存储已计算的子问题解来避免重复计算。与递归实现相比，动态规划消除了大量的重复计算，将时间复杂度从指数级O(2^n)降低到线性级O(n)。

算法步骤

  1. 确定边界条件：F(0) = 0, F(1) = 1
  2. 创建数组或变量存储已计算的斐波那契数
  3. 从小到大（自底向上）计算每个斐波那契数：F(i) = F(i-1) + F(i-2)
  4. 返回F(n)作为结果

  2. 自底向上计算

三、矩阵链乘法

矩阵链乘法

矩阵链乘法问题是指在给定一系列矩阵的情况下，找到一种最优的括号化方式，使得矩阵相乘的计算次数最少。该问题可以通过动态规划高效解决。

时间复杂度:最佳 O(n³) | 平均 O(n³) | 最差 O(n³) 空间复杂度:O(n²)

应用场景/前提条件
  • 动态规划
  • 矩阵乘法
  • 二维动态规划表

算法讲解

介绍

矩阵链乘法（Matrix Chain Multiplication）是一个经典的动态规划问题，给定一系列矩阵，找出相乘的最佳顺序以最小化总的基本运算次数。由于矩阵乘法满足结合律（即(A×B)×C = A×(B×C)），不同的计算顺序会导致相同的结果，但计算量可能差异巨大。

问题定义

给定n个矩阵的链：A₁×A₂×…×Aₙ，其中矩阵Aᵢ的维度为pᵢ₋₁×pᵢ（即行数×列数），目标是找到一种最优的计算顺序，实现完成整个矩阵链的乘法运算所需的标量乘法次数最少。

矩阵乘法的计算代价可以用以下方式定义：两个矩阵A(m×n)和B(n×p)相乘，需要进行m×n×p次标量乘法运算。例如，计算一个2×3矩阵和一个3×4矩阵的乘积，需要2×3×4=24次乘法运算。

不同的乘法顺序会产生截然不同的计算代价。比如三个矩阵A₁(10×100)、A₂(100×5)和A₃(5×50)的乘法：

  • 按照(A₁×A₂)×A₃的顺序：A₁×A₂需要10×100×5=5000次乘法，结果是一个10×5的矩阵；然后与A₃相乘需要10×5×50=2500次乘法，共需要7500次乘法运算。
  • 按照A₁×(A₂×A₃)的顺序：A₂×A₃需要100×5×50=25000次乘法，结果是一个100×50的矩阵；然后与A₁相乘需要10×100×50=50000次乘法，共需要75000次乘法运算。

通过上述例子我们发现，选择不同的计算顺序可能导致运算次数相差十倍以上。随着矩阵数量的增加，可能的括号方案数量呈指数级增长，穷举所有可能的顺序基本不可行。

矩阵链乘法问题的挑战在于，对于n个矩阵的乘积，有卡特兰数量级的不同括号方案。使用动态规划，我们可以在O(n³)的时间复杂度内找到最优解。

算法步骤

  1. 创建一个表dp[i][j]存储从第i个矩阵到第j个矩阵的最小计算代价
  2. 初始化：对角线元素dp[i][i]设为0（单个矩阵无需乘法）
  3. 填表：自底向上计算所有子链的最优解
    • 对于每个长度L（从2到n）
    • 枚举起点i（范围为1到n-L+1）
    • 计算终点j = i+L-1
    • 对于每个可能的分割点k（从i到j-1）
      • 计算代价：dp[i][k] + dp[k+1][j] + p[i-1]×p[k]×p[j]
      • 更新dp[i][j]为所有可能分割点中的最小值
  4. 返回dp[1][n]作为最终结果

  2. 最优计算顺序是((A₁A₂)A₃)。计算A₁A₂需要10×30×5=1500次乘法，得到一个10×5的矩阵；然后与A₃相乘需要10×5×60=3000次乘法。总共需要1500+3000=4500次基本乘法运算。
  3. 需要额外使用一个split数组记录每个区间[i,j]的最优分割点k。在填表过程中，当找到更优的代价时，更新split[i][j]=k。然后使用递归方式，根据split数组构建最优括号方案：如果i=j，返回单个矩阵；否则，分别构建左右子链，再合并。

最长公共子序列 (LCS)

最长公共子序列是指两个序列共同拥有的、保持元素相对顺序的最长子序列。此问题广泛应用于生物信息学、文本相似度分析等领域，是经典的动态规划问题。

应用场景/前提条件

  • 字符串处理

算法讲解

介绍

最长公共子序列（Longest Common Subsequence, LCS）是一个经典的动态规划问题，目标是找出两个序列中共同出现的最长子序列的长度。子序列是指从原序列中删除某些元素（也可以不删除）后得到的新序列，但保持剩余元素的相对顺序不变。

问题定义

给定两个序列X = [x₁, x₂, …, xₘ]和Y = [y₁, y₂, …, yₙ]，找出X和Y的最长公共子序列Z = [z₁, z₂, …, zₖ]，使得Z是X和Y的子序列且长度k最大。

与子串不同，子序列不要求元素连续，只需保持相对顺序。例如，对于序列”ABCBDAB”和”BDCABA”，它们的最长公共子序列是”BCBA”，长度为4。

算法步骤

  1. 创建一个二维数组dp，其中dp[i][j]表示序列X的前i个元素与序列Y的前j个元素的LCS长度
  2. 初始化边界：dp[i][0] = dp[0][j] = 0（空序列的LCS长度为0）
  3. 自底向上填充dp表：
    • 如果x_i = y_j，则dp[i][j] = dp[i-1][j-1] + 1（匹配成功，LCS长度加1）
    • 如果x_i ≠ y_j，则dp[i][j] = max(dp[i-1][j], dp[i][j-1])（取两种可能的最大值）
  4. 返回dp[m][n]作为最终结果

五、背包问题 (0-1 Knapsack)

背包问题 (0-1 Knapsack)

背包问题是一个经典的动态规划问题，给定一组物品，每种物品都有自己的重量和价值，在限定的总重量内，选择若干物品使得物品的总价值最大化。

时间复杂度:最佳 O(n×W) | 平均 O(n×W) | 最差 O(n×W) 空间复杂度:O(n×W)

应用场景/前提条件
  • 动态规划
  • 决策问题
  • 最优化理论

算法讲解

介绍

背包问题（Knapsack Problem）是一类经典的组合优化问题，基本形式是：给定一组物品，每个物品有自己的重量和价值，在限定的总重量内，选择总价值最大的物品。

问题定义

最常见的背包问题是0-1背包问题（0-1 Knapsack Problem）：有n个物品，第i个物品的重量为w[i]，价值为v[i]。现有一个容量为W的背包，问如何选择物品放入背包，在物品总重量不超过背包容量的前提下，物品总价值最大。这里的”0-1”表示每个物品要么完整地放入背包（选择1），要么不放入背包（选择0），不能只放入一个物品的部分或放入多次。

除了0-1背包问题外，还有完全背包问题（物品可重复选择）、多重背包问题（每种物品有数量限制）和分数背包问题（物品可部分选择）等变种。

动态规划是解决背包问题的经典方法，通过构建状态转移方程，可以在多项式时间内求解这类NP完全问题。

算法步骤

以0-1背包问题为例，基于动态规划的解决步骤如下：

  1. 定义状态：dp[i][j]表示考虑前i个物品，背包容量为j时的最大价值
  2. 初始化：
    • dp[0][j] = 0（没有物品时，最大价值为0）
    • dp[i][0] = 0（背包容量为0时，最大价值为0）
  3. 状态转移方程：
    • 如果第i个物品的重量w[i]大于当前背包容量j，则无法放入背包：
      dp[i][j] = dp[i-1][j]
    • 如果可以放入，则可以选择放或不放，取两者最大值：
      dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])
  4. 最终结果：dp[n][W]就是最大价值

假设有以下物品：

  • 物品1：重量3，价值4
  • 物品2：重量4，价值5
  • 物品3：重量5，价值6
  • 背包容量：10

步骤1：创建并初始化DP表

  • 行表示考虑的物品数量（0到3）
  • 列表示背包容量（0到10）
  • dp[0][j]和dp[i][0]均为0

步骤2：填充DP表

  • 对于物品1（i=1）：
    • 当j < 3时，放不下物品1，dp[1][j] = dp[0][j] = 0
    • 当j >= 3时，可以选择放或不放，dp[1][j] = max(dp[0][j], dp[0][j-3] + 4)
    • 例如，dp[1][3] = max(0, 0+4) = 4
  • 对于物品2（i=2）：
    • 当j < 4时，放不下物品2，dp[2][j] = dp[1][j]
    • 当j >= 4时，dp[2][j] = max(dp[1][j], dp[1][j-4] + 5)
    • 例如，dp[2][7] = max(dp[1][7], dp[1][3] + 5) = max(4, 4+5) = 9

算法讲解

介绍

硬币找零问题是动态规划的经典应用场景之一，核心思想是求解如何用给定面值的硬币组合出指定金额，并且使用的硬币数量最少。这个问题在计算机科学与数学领域都有广泛应用，是理解动态规划思想的绝佳例子。

硬币找零问题可以通过动态规划高效求解，时间复杂度为O(n×m)，其中n是目标金额，m是硬币面值的种类数。该问题体现了最优子结构与重叠子问题这两个动态规划的关键特性。

问题定义

给定不同面值的硬币（无限量）和一个总金额，问能凑成总金额的最少硬币个数是多少。例如，硬币面值为[1, 2, 5]，总金额为11时，最少需要3个硬币（5+5+1=11）。如果不能凑出该金额，则返回-1。

硬币找零问题是完全背包问题的一种特例，每种硬币可以重复使用多次。

七、编辑距离

编辑距离

编辑距离算法计算将一个字符串转换为另一个字符串所需的最少操作次数，操作包括插入、删除和替换字符。该算法广泛应用于拼写检查、DNA序列比较和自然语言处理领域。

时间复杂度:最佳 O(mn) | 平均 O(mn) | 最差 O(mn) 空间复杂度:O(mn)

应用场景/前提条件
  • 动态规划
  • 字符串处理
  • 序列比较

算法讲解

介绍

编辑距离（Edit Distance）是计算机科学中的一个基础概念，也被称为Levenshtein距离，用来衡量两个字符串之间的相似度。编辑距离是自然语言处理和生物信息学等领域的核心算法基础。

编辑距离问题是动态规划的典型应用，时间复杂度是O(m×n)，其中m和n分别是两个字符串的长度。这个问题完美体现了动态规划的最优子结构与重叠子问题特性。

问题定义

将一个字符串转换成另一个字符串所需的最少操作数，操作包括插入、删除和替换字符。

例如，将”horse”转换为”ros”的编辑距离为3：

  • 将’h’替换为’r’：horse -> rorse
  • 删除’r’：rorse -> rose
  • 删除’e’：rose -> ros

算法步骤

  1. 定义状态：dp[i][j]表示将word1的前i个字符转换为word2的前j个字符所需的最少操作数
  2. 确定边界条件：
    • dp[i][0] = i（将word1的前i个字符转换为空串需要删除i次）
    • dp[0][j] = j（将空串转换为word2的前j个字符需要插入j次）
  3. 状态转移方程：
    • 若word1[i-1] == word2[j-1]：dp[i][j] = dp[i-1][j-1]
    • 若word1[i-1] != word2[j-1]：dp[i][j] = min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1]) + 1
      • dp[i-1][j-1] + 1：替换操作
      • dp[i-1][j] + 1：删除操作
      • dp[i][j-1] + 1：插入操作
  4. 最终答案：dp[m][n]

算法详解

以一个具体的例子详细说明动态规划的过程：

将字符串”horse”转换为”ros”

步骤1：创建并初始化DP表

  • dp[i][j]表示将”horse”的前i个字符转换为”ros”的前j个字符所需的最少操作数
  • 初始化边界：
    • dp[i][0] = i（将任意字符串变成空串需要删除所有字符）
    • dp[0][j] = j（将空串变成任意字符串需要插入所有字符）

步骤2：填充DP表

对于dp[1][1]，比较’h’和’r’：

  • 不相等，取三种操作的最小值 + 1：
    • 替换：dp[0][0] + 1 = 1
    • 删除：dp[0][1] + 1 = 2
    • 插入：dp[1][0] + 1 = 2
  • 因此，dp[1][1] = 1

对于dp[1][2]，比较’h’和’o’：

  • 不相等，取三种操作的最小值 + 1：
    • 替换：dp[0][1] + 1 = 2
    • 删除：dp[0][2] + 1 = 3
    • 插入：dp[1][1] + 1 = 2
  • 因此，dp[1][2] = 2

依此类推，完整填充整个表格…

步骤3：完整的DP表

步骤4：获取结果

  • 最少操作数为dp[5][3] = 3

步骤5：（可选步骤）回溯找出操作序列
从dp[5][3]开始回溯：

  1. dp[5][3] = dp[4][3] + 1，说明删除了字符’e’
  2. dp[4][3] = dp[3][2] + 1，说明替换了字符’s’为’s’（实际上是相等的，但从表格来看是dp[3][2] + 1）
  3. dp[3][2] = dp[2][1] + 1，说明删除了字符’r’
  4. dp[2][1] = dp[1][0] + 1，说明替换了字符’o’为’r’
  5. dp[1][0] = dp[0][0] + 1，说明删除了字符’h’

根据回溯，操作序列是：替换’h’为’r’，删除’r’，删除’e’。

核心特性

  • 最优子结构：问题的最优解包含子问题的最优解
  • 重叠子问题：相同的子问题会被多次计算
  • 时间复杂度：O(m×n)，其中m和n是两个字符串的长度
  • 空间复杂度：O(m×n)，需要一个二维数组来存储中间状态
  • 适用范围：字符串相似度计算、拼写检查、DNA序列比对

优缺点

优点
  • 可以处理任意长度和内容的字符串
  • 广泛适用于文本处理、生物信息学等领域

这里准备了一些测试题，方便大家判断自己的掌握情况：

  1. 编辑距离的状态转移方程是什么？为什么这样定义？
  2. 当两个字符串完全相同时，编辑距离是多少？为什么？
  3. 优化空间复杂度的关键思路是什么？
  4. 如何修改算法来处理不同操作有不同代价的情况？

测验答案

  1. 状态转移方程是：若字符相同，dp[i][j] = dp[i-1][j-1]；若字符不同，dp[i][j] = min(dp[i-1][j-1], dp[i-1][j], dp[i][j-1]) + 1。表示当前位置的编辑距离等于替换、删除、插入三种操作的最小值再加1，如果字符相同则不需要操作。
  2. 当两个字符串完全相同时，编辑距离为0，因为不需要任何操作就能将一个字符串转换为另一个。
  3. 使用滚动数组，只保留当前行和前一行的数据，因为计算dp[i][j]只依赖于三个位置：左边、上方和左上方。甚至可以进一步优化为只使用一维数组加一个临时变量。
  4. 修改算法处理不同操作代价：在状态转移方程中，加上相应操作的代价。例如，dp[i][j] = min(dp[i-1][j-1] + replaceCost, dp[i-1][j] + deleteCost, dp[i][j-1] + insertCost)。
